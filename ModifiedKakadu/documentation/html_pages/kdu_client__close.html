Synopsis

Same as disconnect, but also discards the entire contents of the cache, causing future calls to is_alive and is_active both to return false. It is safe to call this function at any time, regardless of whether or not it has been successfully opened. If disconnect has already been called, and the underlying transport channel has been saved, the present function will not destroy it — that will happen only if a subsequent call to connect cannot use it, if disconnect is called explicitly with keep_transport_open=false, or if the object is destroyed.
