Synopsis

This function is called with a negative job_idx only if the containing object was registered against a queue using the kdu_thread_entity::register_synchronized_job function. Otherwise, the job_idx argument represents the ordinal position at which the job was added to its queue, starting from 0 for the first job in the queue.
