Synopsis

Base class for objects which can be supplied to kdu_client::install_context_translator for use in translating codestream contexts into their constituent codestreams, and translating the view window into one which is appropriate for each of the codestreams derived from a codestream context. The interface functions provided by this object are designed to match the translating functions provided by the kdu_serve_target object, from which file-format-specific server components are derived.

This class was first defined in Kakadu v4.2 to address the need for translating JPX compositing layers and compositing instructions so that cache model statements could be delivered correctly to a server (for stateless services and for efficient re-use of data cached from a previous session). The translator for JPX files is embodied by the kdu_clientx object, which is, in some sense, the client version of kdu_servex. In the future, however, translators for other file formats such as MJ2 and JPM could be implemented on top of this interface and then used as-is, without modifying the kdu_client implementation itself.
